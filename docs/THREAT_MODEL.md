# 脅威モデル

この文書は、拡張機能が扱う情報と主なリスクを整理するためのものです。

## 保護したいもの

- APIトークン
- メール本文
- メール件名、差出人、宛先、プレビュー
- Masked Emailの作成履歴
- ユーザーのブラウザ内設定
- 閲覧中ページのURL

## 信頼境界

### 信頼するもの

- ユーザーのブラウザ
- ユーザーが設定したAPIトークン
- 対象サービスの公式API

### 信頼しないもの

- メールHTML
- メール本文内のリンク
- メール本文内の外部画像
- 任意のWebページ
- 外部CDNやリモートスクリプト

## 主なリスクと対策

### APIトークン漏えい

リスク:

- APIトークンが外部へ送信される
- コミットやログに混入する
- 過剰な権限を持つトークンを使う

対策:

- APIトークンはローカルストレージに保存する
- 開発者のサーバーへ送信しない
- READMEと設定画面で必要権限を説明する
- リポジトリへ秘密情報をコミットしない

### メールHTMLによる攻撃

リスク:

- スクリプト実行
- フォーム送信
- `javascript:` リンク
- 悪意ある外部画像やトラッキング
- 固定幅HTMLによるUI崩れ

対策:

- 危険なタグと属性を削除する
- リンクURLを検証する
- 外部画像の表示設定を追加する
- HTML表示を無効化できる設定を追加する
- 本文表示領域をスクロール可能にしてUI全体を壊さない

### 過剰なAPI利用

リスク:

- 短すぎる間隔でAPIを呼び出す
- 対象サービスに不要な負荷をかける

対策:

- チェック間隔に下限を設ける
- 取得件数に上限を設ける
- 手動更新と定期更新の挙動を分ける

### 誤解を招く公開

リスク:

- 公式アドオンと誤解される
- 対象サービスへサポート問い合わせが行く

対策:

- 非公式であることを明記する
- 公式ロゴの丸コピーを避ける
- サポート方針を明記する
